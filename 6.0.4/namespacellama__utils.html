◆ LogLevel

enum llama_utils::LogLevel

Enum representing different log levels for controlling log verbosity.

This enum defines the severity levels of logs that can be used to control which log messages should be displayed. The levels are ordered from most severe to least severe. Only logs at or above the current log level will be shown.

Enumerator
ERROR 

Log level for error messages. Only critical errors should be logged.

WARN 

Log level for warning messages. Indicate potential issues that are not critical.

INFO 

Log level for informational messages. General runtime information about the system's state.

DEBUG 

Log level for debug messages. Used for detailed information, mainly for developers.

void llama_utils::apply_eog_logit_biases ( llama_msgs::msg::SamplingConfig & sampling_config,
const llama_vocab * vocab,
const llama_context * ctx )

Applies EOG (End of Generation) logit biases to sampling configuration.

This function handles the configuration of logit biases for EOG tokens based on the ignore_eos setting. It collects all EOG tokens from the vocabulary and adds them to the logit_bias_eog field, then conditionally applies them to the active logit_bias field.

Parameters
sampling_configThe sampling configuration to modify (passed by reference).
vocabThe vocabulary to check for EOG tokens.
ctxThe llama context for token conversion.
